# Чеклист сдачи

## Что должно быть готово

- Видео длиной до 3:00.
- Все 4 участника представлены голосом или лицом.
- Репозиторий с кодом и понятным `README`.
- Краткое описание проекта на 1 страницу A4.
- `requirements.txt` с зафиксированными версиями.
- Список авторов и вкладов.

## Что показать в видео

- Постановка задачи и бизнес-ценность.
- Источник данных и схему пайплайна.
- Рабочий прототип: вход → модель → выход.
- Метрики и анализ ошибок.
- Риски, мониторинг, ограничения и следующие шаги.

## Что нельзя забыть

- Не выйти за 3 минуты.
- Не оставить видео без анализа ошибок.
- Не делать проект без финансового контекста.
- Не говорить только про модель без бизнес-эффекта.
